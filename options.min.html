Great for larger amounts of text content, though not yet fully supported by any browser. The portions lacking support are those controlling column breaks, so that e.g. headings do not end up at the bottom separated from content.

Current options are: 2 columns, 3 columns. Columns are automatically collapsed to 1 column for iPhone screen sizes. Ideally used with CSS3 Media Queries so that, e.g. at large screen sizes 3 columns are used, at smaller, 2 columns, etc.

Columns can be nested... try clicking the column toggle to see the columns above nest within!

Matrix Module

<div class="beeMatrix">
     <div>...</div>
     <div>...</div>
     <div>...</div>
</div>

This is when you want something like a matrix with pieces of similar-width content sitting together when possible, then wrapping as needed. The divs immediately within the matrix div are required wrappers; anything else goes.

No widths and heights are necessary on any divs, unless these are filled with long strings of text, in which case widths should be assigned to the enclosing divs, lest they expand across the page.

Currently the default alignment is bottom, but this can be changed to: top, middle, or bottom.

<div class="beeMatrix middle">
     <div>...</div>
</div>

These CSS tables emulate good old html tables (with all the div wrappers to be expected!) Cells can individually be given horizontal and vertical alignment options: left, center, right, top, middle, bottom (default is top left). Tables can be set to auto-expand to fill whatever container they're in by adding the expand option to the table.
