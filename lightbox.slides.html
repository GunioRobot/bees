Experimental integration with ProjectSeven Lightshow Magic and Slides

Note that any usage of ProjectSeven Lightshow Magic requires a license!

Webkit issue: on a page with multiple sliders, when css3 3d transforms apply to elements, the first slider must first be interacted with before the other sliders; if not, all sliders beyond the first slider will advance/ reverse normally, but without easing (until after such time as the first is interacted with).

div.beeFlip causes issue inside div.item:

.beeFlip {
    position: relative;
    -webkit-perspective: 1000;
}
.beeFlip .flip {
    position: absolute;
    top: 0;
    left: 0;
    -webkit-transform-style: preserve-3d;
    -webkit-backface-visibility: hidden;
    -webkit-transition: 0.75s;
}
.beeFlip .front {
    z-index: 5;
}
.beeFlip:hover .front {
    z-index: 5;
    -webkit-transform: rotateY(180deg);
}
.beeFlip .back {
    z-index: 0;
    -webkit-transform: rotateY(-180deg);
}
.beeFlip:hover .back {
    z-index: 10;
    -webkit-transform: rotateY(0deg);
}

Note that relative/ absolute positioning does not cause the problem; easing works perfectly on all sliders with above css rules set, so long as all css3 3d transform rules (bold) are removed.
